The News Server REST API demonstrates an example of API capabilities and available endpoints for querying a mock-up of a basic news-based site where users can post, and interact with, articles.

The project is currently a work in progress, and only working API endpoints, routes, and queries are fully documented below. As such, new endpoints and parameters are likely to become publicly available in the near future.

All APIs are currently unrestricted and accessible to all.

A list of new changes are summarised in the footer along with a roadmap for future planned development.

About

All APIs are currently unrestricted and accessible to all, and a list of endpoints, with information about what can be accessed, and how is available in [The endpoints](endpoints.json).

The DB and api were developed and tested with the following packages:

  • express: ^5.1.0
  • dotenv: ^16.5.0
  • nodemon: ^3.1.10
  • pg: ^8.16.0
  • pg-format": "^1.0.4


The hosted DB is run using the following engine:

  • node: >=6.0.0

Data Dictionary

Articles

Variable Description Type
article_id A unique id for each article, database defined. Integer
Primary Key
title A free-text title of the article, user-defined. Varchar (1000)
GET /api/articles?sort... ... must take the form of column_name:order. The column_name pertains to columns in the articles table and the order can be asc (for ascending) or desc (for descending). The inputs are case insensitive. The return serves an object with the key of articles and the value of an array of article objects. Where no sorting parameters are provided, the articles are sorted by date, in descending order.

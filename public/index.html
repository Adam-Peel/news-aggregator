News Server API Endpoints

The News Server REST API demonstrates an example of API capabilities and available endpoints for querying a mock-up of a basic news-based site where users can post, and interact with, articles.

The project is currently a work in progress, and only working API endpoints are listed below. As such, new endpoints and parameters are likely to become publicly available in the near future.

A list of new changes are summarised in the footer along with a roadmap for future planned development.

Articles

Endpoint Description Example
GET /api/articles serves an object with the key of articles and the value of an array of article objects. The articles are sorted by date, in descending order. 
"exampleQuery": "GET /api/articles",
    "exampleResponse": {
      "articles": [

GET /api/articles?sort... ... must take the form of column_name:order. The column_name pertains to columns in the articles table and the order can be asc (for ascending) or desc (for descending). The inputs are case insensitive. The return serves an object with the key of articles and the value of an array of article objects. Where no sorting parameters are provided, the articles are sorted by date, in descending order.
